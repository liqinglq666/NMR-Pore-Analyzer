# NMR-Pore-Analyzer v2.1

> 这是我围绕低场核磁共振（LF-NMR）T₂ 弛豫谱数据处理建立的孔隙结构分析工具。  
> 在做水泥基材料孔结构分析时，我发现仪器导出的原始 T₂ 数据距离论文中真正可用的孔径分布、孔隙分级、峰值统计和对比图表还有一段距离，因此尝试把这套后处理流程整理成一个相对规范、可复现、可扩展的桌面程序。  
> 这个项目不是单纯为了做界面，而是希望把材料微结构表征中的物理假设、数值计算和科研结果输出串联起来。

---

## 1. 我为什么做这个项目

我在处理水泥基材料 LF-NMR 数据时遇到一个很实际的问题：仪器导出的 T₂ 谱虽然包含了孔隙水弛豫信息，但它本身并不能直接支撑论文中的孔结构分析。真正写论文时，我需要的不只是曲线，而是能够回答以下问题的一组定量结果：

1. 不同配合比或不同龄期下，孔径分布是否发生细化或粗化？
2. 凝胶孔、过渡孔、毛细孔和大孔比例如何变化？
3. 有害孔和无害孔比例是否能够反映材料耐久性差异？
4. T₂ 谱中是否真的存在次峰，还是只是高 T₂ 区间的单调尾部？
5. 主峰、次峰和谷值的位置能否批量、稳定、可复现地提取？
6. 数据能否直接导出为后续论文绘图和结果分析所需的格式？

如果这些步骤全部手动完成，不仅效率低，而且容易在不同样品之间引入不一致的处理规则。尤其是孔隙分级阈值、积分方式、峰值识别规则这些细节，一旦处理不严谨，就会影响后续对微结构演化和材料性能的解释。

因此，我把这个项目定位为一个“研究生科研数据处理工具”：它服务于材料实验数据的规范化处理，也体现我对材料表征、理论转换、数值计算和软件实现之间关系的理解。

这个项目的目标不是做一个复杂的软件系统，而是把 LF-NMR 孔结构分析中最常用、最容易出错、最需要统一规则的部分整理出来，形成一套稳定流程。

| 我的研究需求 | 项目中的实现 |

## 3. 我的理论理解与计算假设

### 3.1 从 T₂ 弛豫时间到孔径的转换

LF-NMR 用于孔结构分析的基础在于：孔隙水的横向弛豫时间 T₂ 与孔隙尺度之间存在联系。对于饱和多孔材料，横向弛豫速率通常可以写成：

```math
\frac{1}{T_2}
= \frac{1}{T_{2,\text{bulk}}}
+ \rho_2 \cdot \frac{S}{V}
+ \frac{D(\gamma G T_E)^2}{12}
```

其中，`T₂` 是横向弛豫时间，`ρ₂` 是表面弛豫率，`S/V` 是孔隙表面积与体积之比。在短回波间隔条件下，如果体相弛豫和扩散弛豫影响相对较小，可以近似认为：

```math
\frac{1}{T_2}
\approx \rho_2 \cdot \frac{S}{V}
= \rho_2 \cdot \frac{F_s}{r}
```

这意味着 T₂ 越长，对应的孔隙尺度通常越大。为了将 T₂ 轴转换为更直观的孔径轴，我在程序中采用如下标定关系：

```math
T_2 = 4.2\,\text{ms}
\quad \Longleftrightarrow \quad
r = 100\,\text{nm}
```

因此：

```math
r\,[\text{nm}]
= \frac{100}{4.2}T_2\,[\text{ms}]
\approx 23.81T_2
```

这个换算关系不是写死在算法深处，而是集中放在 `logic/config.py` 中。这样做是因为我认为科研程序必须保留可校准性：如果后续材料体系、孔隙液性质或表面弛豫率发生变化，应当能够方便地调整转换系数，而不是把默认参数当成不可修改的绝对结论。

### 3.2 我设置的两套孔隙分类体系

为了让结果既能解释微观结构，又能服务耐久性讨论，我在项目中并行设置了两套分类体系。

#### System A：物理形态分类

这套分类主要用于描述孔隙尺度和孔结构形态。

| 孔隙类型 | T₂ 范围 / ms | 孔径范围 / nm | 我的理解 |
|---|---:|---:|---|
| Gel pores | `[0, 0.42)` | `[0, 10)` | 主要对应凝胶孔和非常细小的孔隙 |
| Transition pores | `[0.42, 4.2)` | `[10, 100)` | 处于凝胶孔与毛细孔之间的过渡孔 |
| Capillary pores | `[4.2, 41.7)` | `[100, 1000)` | 与毛细孔相关，通常更影响传输性能 |
| Air-voids | `[41.7, +∞)` | `[1000, +∞)` | 较大孔隙、气孔或明显缺陷孔 |

#### System B：损伤潜势分类

这套分类更多从耐久性和潜在劣化角度理解孔径分布。

| 孔隙类型 | T₂ 范围 / ms | 孔径范围 / nm | 我的理解 |
|---|---:|---:|---|
| Harmless pores | `[0, 0.83)` | `[0, 20)` | 较细孔隙，对传输和损伤影响相对较小 |
| Less-harmful pores | `[0.83, 2.08)` | `[20, 50)` | 中小孔隙，可能对性能产生一定影响 |
| Harmful pores | `[2.08, 8.33)` | `[50, 200)` | 与渗透、离子传输和耐久性劣化更相关 |
| More-harmful pores | `[8.33, +∞)` | `[200, +∞)` | 粗大孔、连通孔或缺陷孔，通常不利于耐久性 |

我保留两套体系，是因为单一分类往往很难同时满足“孔结构形态描述”和“性能风险讨论”。System A 更适合写微观结构演化，System B 更适合写有害孔比例和耐久性分析。

## 4. 我对积分与峰值识别的处理

### 4.1 为什么保留三种积分模式

LF-NMR 反演谱通常是离散谱，但不同仪器或不同导出方式的 T₂ 轴并不完全一样。如果直接默认一种积分方式，可能会造成结果解释上的偏差。因此我保留了三种模式：

| 积分模式 | 我建议的使用场景 | 计算含义 |
|---|---|---|
| Bin Summation | 默认推荐，用于多数仪器导出的离散反演谱 | 对区间内幅值直接求和 |
| Log-domain Integration | 当 T₂ 轴近似对数采样，并希望按连续曲线理解时使用 | 在 `log10(T₂)` 轴上积分 |
| Linear Integration | 仅当 T₂ 轴确实是线性采样时使用 | 在原始 T₂ 轴上积分 |

Bin Summation 的基本形式为：

```math
S_k = \sum_{i \in k} A_i
```

孔隙比例计算为：

```math
\phi_k = \frac{|S_k|}{\sum_j |S_j|}
```

对于 Log-domain 和 Linear 积分，我在程序中加入了分类边界插值。这样做是为了避免一个常见问题：当孔隙分类阈值落在两个采样点之间时，如果只是简单用 mask 截断，就会漏掉一部分面积。这个细节看起来很小，但在多组样品对比时会影响各孔径区间的比例。

### 4.2 为什么要严格识别次峰

在分析 T₂ 谱时，我比较关注一个问题：高 T₂ 区间出现信号并不一定意味着存在真正的第二峰。有时它只是主峰后的长尾。如果把这种单调尾部也识别为次峰，就可能把孔结构粗化程度夸大。

因此，我将主峰定义为 `[0, 10)` ms 区间内的全局最大值：

```math
i_{pri} = \arg\max_{i:T_{2,i}\in[0,10)} A_i
```

而次峰必须是 `(10, 1000]` ms 区间内的严格局部极大值：

```math
A_i > A_{i-1}
\quad \text{and} \quad
A_i > A_{i+1}
```

如果高 T₂ 区间没有严格局部极大值，程序就不会输出次峰。这样虽然更保守，但我认为更符合科研分析中“宁可少解释，也不要过度解释”的原则。

### 4.3 谷值与 fallback 机制

当主峰和次峰同时存在时，我会进一步寻找两峰之间的局部极小值作为分割边界。如果存在真实谷值，则用它划分主峰区和次峰区；如果不存在真实谷值，则使用 `T₂ = 10 ms` 作为 fallback boundary，并在结果中明确标记。

这种设计的意义在于：即使程序为了统计需要给出分割结果，也必须告诉使用者这个分割是由真实谱形决定的，还是由规则兜底得到的。对科研数据而言，这种透明性比单纯给出一个看似完整的结果更重要。

## 5. 我是如何组织代码的

我没有把所有代码写在一个脚本里，而是把科学计算逻辑、峰值处理、结果导出、绘图和界面分开。这样做主要是为了让项目更容易维护，也便于后续继续扩展。

```text
NMR-Pore-Analyzer/
├── main.py                    # 程序入口
├── requirements.txt           # 运行依赖
├── requirements-dev.txt       # 测试依赖
├── logic/
│   ├── config.py              # 物理常数、孔隙阈值、列名别名、版本信息
│   ├── analyzer.py            # 数据读取、清洗、孔径转换、孔隙分类
│   ├── peak_processor.py      # 主峰、次峰、谷值识别
│   └── exporter.py            # Excel 科研结果表导出
├── ui/
│   ├── main_window.py         # PySide6 主界面
│   ├── main_window_safe.py    # 线程生命周期安全封装
│   └── plot_canvas.py         # Matplotlib 绘图画布
└── tests/
    ├── conftest.py            # pytest 路径配置
    └── test_core_logic.py     # 核心逻辑测试
```

整体数据流如下：

```mermaid
flowchart LR
    A[仪器导出数据] --> B[读取 xlsx/xls/csv]
    B --> C[识别 T₂ 列与样品列]
    C --> D[数值化与异常值清洗]
    D --> E[合并重复 T₂ bin]
    E --> F[T₂ 转换为孔径]
    F --> G[孔隙分级积分]
    F --> H[峰值与谷值识别]
    G --> I[统计表与曲线数据]
    H --> I
    I --> J[Excel 结果表]
    I --> K[高分辨率图像]
```

这种结构虽然比单文件脚本麻烦一些，但更符合我对科研程序的理解：核心计算逻辑应该可以独立测试，界面只是调用这些逻辑，而不是把科学判断隐藏在界面按钮后面。

## 7. 输入数据格式

输入文件至少需要包含一列 T₂ 时间轴和一列信号幅值列。如果有多列样品数据，程序会把每一列当作一个独立样品进行批量分析。

示例：

| T2(ms) | Mix-1 | Mix-2 | Mix-3 |
|---:|---:|---:|---:|
| 0.01 | 12.3 | 11.9 | 13.1 |
| 0.02 | 15.6 | 14.8 | 15.2 |
| 0.05 | 18.2 | 17.1 | 19.0 |
| ... | ... | ... | ... |

程序支持常见英文和中文表头识别：

- T₂ 列：`T2`、`T2(ms)`、`T₂(ms)`、`time(ms)`、`relaxation time`、`弛豫时间`、`弛豫时间/ms`；
- 信号列：`amplitude`、`signal`、`intensity`、`dv/dr`、`幅值`、`信号强度`、`孔隙度`、`增量孔隙度`。

支持文件格式：

- `.xlsx`：使用 `openpyxl` 读取；
- `.xls`：使用 `xlrd` 读取；
- `.csv`：优先使用 `utf-8-sig`，失败后自动尝试 `gbk`，用于兼容中文仪器导出数据。

## 8. 输出结果

程序会导出四张 Excel 结果表：

| Sheet | 内容 |
|---|---|
| `Summary_Peak_Statistics` | 主峰、次峰、谷值、主峰/次峰面积比例 |
| `Pore_Classification_Ratios` | System A 与 System B 各类孔隙比例 |
| `Cumulative_Curve_Data` | 各样品孔径-累积分布曲线数据 |
| `Differential_Curve_Data` | 各样品孔径-增量信号分布数据 |

图像支持导出为：

- PNG；
- PDF；
- SVG。

这些输出主要服务于论文图表绘制、实验组间对比、龄期演化分析和微观结构讨论。

## 11. 我目前对这个工具边界的认识

为了避免把工具结果过度解释，我也明确保留以下边界说明：

1. 默认采用 `4.2 ms ↔ 100 nm` 的标定关系，不同材料体系应结合实际表面弛豫率进行校准；
2. 程序默认保留正 T₂ 与正信号幅值，NaN 和非正值会在清洗阶段删除；
3. 对大多数 LF-NMR 仪器反演谱，我更推荐使用 Bin Summation；Linear Integration 只适合线性采样 T₂ 轴；
4. 峰值分割是一种定量描述方法，不能脱离原始谱图、配合比、养护条件和其他微观测试结果单独解释；
5. 本工具强调透明、可复现和可审查，它不能替代研究者对材料机理的判断。
